# Security Policy

## Supported Versions

Currently, only the latest version of the Notesify application is supported with active security updates. 

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

Security is a top priority for Notesify. If you discover a security vulnerability, please do NOT open a public GitHub issue. 

Instead, please send an email directly to **badalsahani233@gmail.com**.

Please include the following in your email:
- A description of the vulnerability.
- Steps to reproduce the issue.
- Any relevant code snippets, logs, or screenshots.
- Possible impact of the vulnerability.

We will acknowledge your email within 48 hours and provide a timeline for a fix. Once the issue is resolved and deployed, we will publicly acknowledge your contribution in our release notes (if you wish to be credited).

### Scope
This policy applies to:
- The Notesify web application (frontend and backend)
- The Notesify desktop application
- The Notesify landing page
- Any official APIs or services managed under this repository.

Thank you for helping keep Notesify secure!
